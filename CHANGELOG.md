# Журнал изменений

Все заметные изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
а commit messages в проекте следуют Conventional Commits.

## [Не выпущено]

### Добавлено

- Переключатели **«Докупка доступна всегда»** в редакторе тарифа: независимые
  поля `topup_always_available` (обычный трафик) и
  `premium_topup_always_available` (premium-трафик).
- Публичная ссылка инструкции установки `/s/<token>` в блоке ссылок карточки
  пользователя админки — и в Web App, и в Telegram-админке.
- No-op observability seam для плагинов (`bot/infra/observability.py` и хук в
  Plugin API).
- Совместимость с Remnawave 2.8 и versioned-пресеты dev-стенда
  (`deploy/dev`).
- Детали даты сброса трафика в предупреждениях о расходе квоты.
- Одно-командное dev-окружение: цели `Makefile` и devcontainer.
- Политика приватного раскрытия уязвимостей, GitHub issue templates,
  pull request checklist, CODEOWNERS и этот журнал изменений.

### Изменено

- Предложение докупки трафика разблокируется, когда израсходовано не менее
  80% лимита; обычный и premium-трафик гейтятся независимо, в меню бота
  кнопка докупки скрыта, пока предложение заблокировано.
- Весь first-party код фронтенда переведен на TypeScript; архитектурные гейты
  запрещают новые `.js`-файлы и `<script>` без `lang="ts"` в `frontend/src`.
- Все admin/webapp HTTP-ответы описаны типизированными схемами; loose schemas
  в OpenAPI-артефакте больше нет, не-JSON маршруты закреплены явно.
- Версии Node.js закреплены через `engines` (`>=22` в корне и frontend,
  `>=22.12` для docs-site), инструментальные версии выровнены.
- Корневая команда `npm run check` стала полной локальной проверкой и
  зеркалирует `make check`.

### Исправлено

- Восстановлена кнопка докупки трафика в меню Telegram-бота: callback
  `tariff_topup:list` потерял регистрацию при разделении subscription-core.
- Инструкции установки: конфиг конкретной подписки повторно запрашивается со
  свежим short UUID панели после сброса/перевыпуска ссылки подписки.
- Предупреждения о трафике используют даты сброса из панели.
- WebApp-листенер открывается до Telegram startup-обменов, healthcheck-пробы
  контейнеров чаще выполняются на старте.
